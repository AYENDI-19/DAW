FLUJO OPERATIVO

Control de accesos en tiempo real

Paso Acción ¿Qué sucede internamente?
1. Registro Alta rápida desde panel El administrador crea al cliente con 1 correo y contraseña. El sistema envía las credenciales automáticamente por email.
2. Identificación Pase Digital (QR) El cliente visualiza su código QR único (IRON PASS) en su teléfono al acceder.
3. Validación Escaneo instantáneo La API evalúa el estado (`activo`/`suspendido`) y vigencia de la suscripción. Si la membresía está activa, concede acceso.
4. Bloqueo/Acción Auditoría automática Si está suspendido por falta de pago, deniega la entrada mostrando el motivo y registra el intento fallido.
INTELIGENCIA DE NEGOCIO

Previene bajas con la Alarma de Churn

La retención inteligente permite actuar proactivamente antes de perder al cliente. El sistema gestiona esto mediante tres pilares:

  • 🔍

    Umbral programable flexible

    Ajusta el número de días ausentes permitidos (ej. 10 días) según las dinámicas de tu centro.

  • ⚡

    Segmentación instantánea

    El sistema genera una lista limpia ordenada por riesgo (días sin asistir) para el equipo de fidelización.

  • ✉️

    Recuperación reactiva (1-Clic)

    Permite lanzar alertas de contacto directo o campañas motivacionales directamente al correo electrónico.

PASO A PASO (DEVOPs)

Puesta en Marcha: Guía de Producción

  • 1

    Fase BD: Creación y Semilla (Clever Cloud)

    Se crea el addon MySQL. Ejecutamos de forma local el script de migración y siembra seed.py apuntando a la URI remota para estructurar las tablas (usuarios, clientes, accesos, rutinas) e insertar datos iniciales.

  • 2

    Fase API: Variables de Entorno (Render)

    Creamos el Web Service apuntando al backend del repo. Configuramos las variables esenciales: DATABASE_URL (enlace MySQL), SECRET_KEY, ALGORITHM (cifrado JWT) y PROD_ORIGIN (CORS hacia Vercel).

  • 3

    Fase Web: Reverse Proxy y CORS (Vercel)

    Desplegamos el frontend. Para evitar problemas de CORS y bloqueos por HTTP mixto (Mixed Content), creamos un archivo vercel.json en la raíz que actúa de proxy inverso redireccionando solicitudes de forma transparente.

💡 El Secreto del vercel.json

Al indicarle a Vercel que reescriba la ruta /api/* hacia el dominio de Render, el navegador cree que consume datos del mismo origen. ¡Adiós a los errores de CORS en el cliente! 

{
  "rewrites": [
    {
      "source": "/api/:path*",
      "destination": "https://backend.render.com/api/:path*"
    },
    {
      "source": "/(.*)",
      "destination": "/"
    }
  ]
}
